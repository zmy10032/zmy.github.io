北太天元学习19-谷歌搜索算法(page rank algorithm)简介

我们假设有4个网页
在初始时刻，网页1有很多人在浏览，其他网页无人浏览
所有浏览网页的人记作1
每隔一个单位时间都有部分人转移到别的网页
定义在时间n时的态为
p_n = [p_{1, n}, p_{2, n}, p_{3, n}, p_{4, n}]
其中p_{i, n}表示在n时刻处在网页i上的浏览者的比例
所以初始态p_0 = [1, 0, 0, 0]
p_{n + 1} 和 p_{n} 之间的关系是建模的重点：
p_{n + 1} = A * p_{n}
其中矩阵 A 称为状态转移矩阵
则p_{1, n+1} = A(1, 1) * p_{1, n} + A(1, 2) * p_{2, n} + A(1, 3) * p_{3, n} + A(1, 4) * p_{4, n}
如果考虑有人不想浏览网页了，可以增加第五个“网页”，到第五个“网页”就表示下网了。

我们可以这样来建模：

  1. 如果第i个网页有k个其他网页的链接，那么转移到它们的概率是1/(k + 1)
  2. 如果该网页没有其他网页的链接，那么浏览者会随机选择一个网站

给一个具体的例子：
网页1拥有链接2、4，网页2没有链接，网页3有链接1，网页4有链接2、3
那么 A =
1/3 1/4 1/2 0
1/3 1/4 0 1/3
0 1/4 1/2 1/3
1/3 1/4 0 1/3
x_0 = [1; 0; 0; 0]

脚本代码如下：

A = [1/3 1/4 1/2 0
1/3 1/4 0 1/3
0 1/4 1/2 1/3
1/3 1/4 0 1/3];
初始p = [1; 0; 0; 0];
N = 20;
p = zeros(4, N);
p(:, 1) = 初始p;
for n = 1:N
p(:, n + 1) = A * p(:, n);
end
hold on
for j = 1:4
plot(1:N, p(j, :), ‘LineWidth’, 3);
str(j) = sprintf(“网页%d”, j);
label_str(j) = sprintf(“网页%d最终占比%d%%”, j, 100*p(:, end));
end
hold off;
legend(str);
figure(2)
pie(p(:, end), label_str);

  • str(j) = sprintf(......) 将输出写入到字符串中，而不是直接输出到屏幕
  • figure(2) 创建编号为 2 的图形窗口。如果 2 号窗口已经存在，它将霸占这个窗口
  • pie(p(:end), label_str) 画一个饼图，把label_str里的字符串按顺序放在饼各部分旁边

视频最后有个发现：最终态是稳定的，并且这个态 ppp 和 A 最大特征值对应的特征向量 pp 成比例:
ppp = pp/norm(pp, 1)

  • norm(pp, 1) 得到 pp 的长度

但原理是啥并没有解释

==此算法可能在足球比赛、投票等题目中出现==
